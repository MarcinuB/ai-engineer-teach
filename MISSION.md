# Mission

## Why

Marcin wants to become a working AI engineer who can ship production LLM-powered features. He has 20 years of software engineering experience across multiple languages and understands systems, APIs, and architecture deeply — but needs to close the gap on Python and modern AI engineering patterns to transition into this role.

## Success Looks Like

- Shipping 5 real projects from the article's roadmap (structured output extractor, RAG assistant, tool-using workflow, stateful agent, deployed mini-product)
- Able to design and build an LLM-powered feature end-to-end: from prompt design to deployed API with observability
- Can evaluate system quality with metrics (quality pass rate, latency, cost-per-run) — not just vibes

## Constraints

- Python is the weakest link — lessons should teach Python idioms only as needed, not from scratch
- 90-day timeline (loosely, from the article's roadmap)
- Strong engineering foundation means no need to explain HTTP, APIs, JSON, async, debugging — skip all of that

## Out of Scope

- Training models from scratch
- GPU optimization or CUDA
- Fine-tuning or custom embedding model training
- Theoretical deep learning or ML math
