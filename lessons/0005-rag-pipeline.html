AI Engineering Lesson 05

RAG Pipeline

Retrieve relevant documents, inject them as context, let the LLM answer from your data.

What changes from lesson 04

In lesson 04 you built the retrieval step: embed a query, find the closest documents. RAG adds one thing — you pass those documents to an LLM as context and ask it to answer from them.

RAG = Lesson 04's search + one new step: inject retrieved docs into the LLM's system prompt.

Build it — step by step

Create invoice-extractor/lesson_005/rag.py. Also create a docs/ subfolder — that's where your source documents will live.

  • Imports and clients

    Two clients, two models — one for embeddings, one for chat. Common pattern in RAG apps.

    rag.py
    import math
import os
from pathlib import Path
from openai import OpenAI

client = OpenAI(base_url="http://localhost:11434/v1", api_key="ollama")
EMBED_MODEL = "nomic-embed-text"

    🐍 Python nugget — pathlib.Path

    pathlib.Path is the modern way to work with file paths — object-oriented, cross-platform, and composable. Instead of os.path.join("docs", "inv.txt") you write Path("docs") / "inv.txt". Key methods: .read_text(), .write_text(), .glob("*.txt"), .stem (filename without extension), .parent.

  • Load and embed documents

    Read every .txt file from the docs/ folder and embed its content. Store the text alongside the embedding so you can return it to the LLM later.

    rag.py
    def embed(text: str) -> list[float]:
    response = client.embeddings.create(model=EMBED_MODEL, input=text)
    return response.data[0].embedding

def cosine_similarity(a: list[float], b: list[float]) -> float:
    dot   = sum(x * y for x, y in zip(a, b))
    norm_a = math.sqrt(sum(x * x for x in a))
    norm_b = math.sqrt(sum(y * y for y in b))
    return dot / (norm_a * norm_b)

# Load every .txt file from the docs/ folder
DOCS_DIR = Path(__file__).parent / "docs"

docs = [p.read_text() for p in sorted(DOCS_DIR.glob("*.txt"))]
doc_embeddings = [embed(doc) for doc in docs]

print(f"Loaded {len(docs)} documents.")

    🐍 Python nugget — Path.glob() and Path.read_text()

    Path.glob("*.txt") returns an iterator of all .txt files in the directory. Order is filesystem-dependent (not guaranteed alphabetical), so wrap in sorted() to get consistent ordering.

    path.read_text() reads the whole file as a string. Default encoding is platform-dependent — in production, always pass encoding="utf-8" explicitly to avoid surprises on Windows.

  • Search function

    Same as lesson 04, with a score threshold added — so low-relevance documents don't get passed to the LLM as fake context.

    rag.py
    def search(query: str, top_k: int = 2, threshold: float = 0.5) -> list[tuple[str, float]]:
    query_vec = embed(query)
    scores = [cosine_similarity(query_vec, dv) for dv in doc_embeddings]
    ranked = sorted(zip(docs, scores), key=lambda p: p[1], reverse=True)
    return [(doc, score) for doc, score in ranked[:top_k] if score >= threshold]
  • Context builder — the new piece

    Takes retrieved documents and formats them into a string you'll inject into the system prompt. Numbered citations let the LLM refer back to specific sources.

    rag.py
    def build_context(results: list[tuple[str, float]]) -> str:
    return "\n\n".join(
        f"[Document {i + 1}]\n{doc}"
        for i, (doc, score) in enumerate(results)
    )

    🐍 Python nugget — enumerate() and str.join()

    enumerate(iterable) yields (index, value) pairs: for i, val in enumerate(["a","b","c"]) gives (0,"a"), (1,"b"), (2,"c"). Replaces for i in range(len(...)) when you need both index and value. Start from 1 with enumerate(items, start=1).

    "\n\n".join(iterable) concatenates strings with a double newline between each. The separator goes between items, never at the start or end — so no trailing newline. Passing a generator expression (not a list) is fine: join() accepts any iterable.

  • RAG query function — ties it all together

    Retrieve relevant docs, inject them as context, send to the LLM. The system prompt instructs the model to answer only from the provided documents and say so if the answer isn't there.

    rag.py
    def rag_query(question: str) -> str:
    results = search(question)

    if not results:
        return "I couldn't find any relevant documents to answer that question."

    context = build_context(results)

    system_prompt = f"""You are an invoice assistant. Answer the user's question using ONLY \
the documents provided below. If the answer is not in the documents, say so explicitly \
— do not use your training knowledge.

DOCUMENTS:
{context}"""

    response = client.chat.completions.create(
        model=CHAT_MODEL,
        messages=[
            {"role": "user", "content": question},
        ],
        extra_body={"system": system_prompt},
    )
    return response.choices[0].message.content

    Passing the system prompt

    Different providers handle the system prompt differently. The standard OpenAI way is to add {"role": "system", "content": system_prompt} as the first message in the messages list — not as an extra_body field. Try this if extra_body doesn't work with your provider: 

    messages=[
    {"role": "system", "content": system_prompt},
    {"role": "user",   "content": question},
],

Challenges

1. Show which documents were retrieved

Print the document filenames and scores before the answer — so you can see why the LLM gave that response. Modify rag_query to also return the retrieved sources, not just the answer.

2. Add a 4th document about something unrelated

Create docs/company_picnic.txt with a short event announcement. Ask "When is the company picnic?" — it should be retrieved and answered correctly. Then ask an invoice question and verify the picnic doc scores low and doesn't pollute the context.

3. Try a hallucination

Remove the "only use the documents" instruction from the system prompt. Ask a question that isn't in any document. Watch the LLM confidently invent an answer. Then put the instruction back. This is the core reason the system prompt matters in RAG.

What's next

  • Chunking: long documents need to be split into chunks before embedding — the full file may exceed the embedding model's token limit, and smaller chunks give more precise retrieval
  • Vector stores: ChromaDB or Qdrant replace the in-memory list — they persist embeddings to disk and support millions of documents
  • Multi-turn RAG: maintain a conversation history so follow-up questions work ("what about the other one?")

What you've now built

A working RAG system — project #2 from the mission. It's basic (in-memory, no chunking, no persistence) but the architecture is real. Every production RAG system is this loop with better infrastructure around it.
